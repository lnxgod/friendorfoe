# Friend or Foe — شناسایی هواپیما و پهپاد در زمان واقعی

[![Android Build](https://github.com/lnxgod/friendorfoe/actions/workflows/android-build.yml/badge.svg)](https://github.com/lnxgod/friendorfoe/actions/workflows/android-build.yml)

**گوشی خود را به سمت آسمان بگیرید. بدانید آن بالا چه خبر است.**

Friend or Foe یک اپلیکیشن اندروید متن‌باز با قابلیت **نصب-و-شروع** است که با استفاده از واقعیت افزوده، هواپیماها و پهپادها را در زمان واقعی شناسایی می‌کند. این اپلیکیشن داده‌های ترانسپوندر ADS-B، پخش FAA Remote ID پهپاد، تحلیل سیگنال WiFi و تشخیص بصری روی دستگاه را ترکیب می‌کند تا برچسب‌های شناور روی نمای دوربین قرار دهد — به شما می‌گوید چه چیزی بالای سرتان پرواز می‌کند، چه کسی آن را اداره می‌کند، کجا می‌رود و آیا دوست است یا دشمن. بدون حساب کاربری، بدون ثبت‌نام، بدون کلید API — فقط نصب کنید و شروع کنید.

این پروژه **با هوش مصنوعی ساخته شد** — نه فقط یکی، بلکه همه آن‌ها. Claude کد را نوشت، Grok طراحی را شکل داد، Codex امنیت را بررسی کرد و Gemini به انتخاب پشته فناوری کمک کرد. توسط [GAMECHANGERSai](https://gamechangersai.org) منتشر شد تا نشان دهد وقتی هوش مصنوعی با خلاقیت انسانی ملاقات می‌کند چه چیزی ممکن است. برای تاریخچه نسخه‌ها [CHANGELOG](CHANGELOG.md) را ببینید.

### دوی سرعت ۷۲ ساعته

> **از صفر تا شناسایی تأیید شده هواپیما و پهپاد روی یک دستگاه واقعی — در کمتر از ۷۲ ساعت.**
>
> در ۱۲ مارس ۲۰۲۵، اولین commit همان شب انجام شد، ۱۳ commit و **بیش از ۸,۵۰۰ خط کد** در حدود ۲ ساعت برنامه‌نویسی جفتی با AI با استفاده از Claude نوشته شد — یک APK آماده ساخت با نمایشگر AR، تشخیص چند منبعی، ادغام سنسور بیزی و نمای نقشه تولید کرد. تا ۱۴ مارس، اپلیکیشن با سایه‌نماهای هواپیما، نشانگرهای نقشه طراحی شده و پرداخت به صورت متن‌باز منتشر شد — مجموعاً **بیش از ۲۲,۰۰۰ خط** در Kotlin، Python و XML.
>
> اپلیکیشن **نصب-و-شروع** است — APK را نصب کنید، مجوزها را بدهید و شناسایی هواپیماها را شروع کنید. بدون ثبت‌نام، بدون کلید، بدون حساب مستقیماً به API های عمومی رایگان ADS-B متصل می‌شود. بک‌اند اختیاری Python فقط غنی‌سازی اضافی اضافه می‌کند (نام شرکت‌های هواپیمایی، اطلاعات مسیر).

## چه کارهایی انجام می‌دهد

- **نمایشگر AR** — نمای زنده دوربین با برچسب‌های شناور رنگی روی هواپیماها و پهپادهای شناسایی شده. برچسب‌ها علامت تماس، نوع هواپیما، ارتفاع و فاصله را نشان می‌دهند.
- **تشخیص چند منبعی** — چهار روش تشخیص مستقل را برای آگاهی جامع آسمان ترکیب می‌کند:
  - داده‌های ترانسپوندر ADS-B (پروازهای تجاری، هوانوردی عمومی)
  - FAA Remote ID از طریق Bluetooth LE (پهپادهای سازگار)
  - تطابق الگوی WiFi SSID (DJI، Skydio، Parrot، بیش از ۱۰۰ سازنده)
  - تشخیص بصری با ML Kit (تشخیص اشیا مبتنی بر دوربین)
- **طبقه‌بندی هوشمند** — همه چیز را در ۱۰ نوع طبقه‌بندی می‌کند: تجاری، هوانوردی عمومی، نظامی، هلیکوپتر، دولتی، اورژانس، باری، پهپاد، وسیله نقلیه زمینی و ناشناخته. تشخیص نظامی از الگوهای علامت تماس، کدهای squawk و پایگاه‌های داده اپراتور استفاده می‌کند.
- **ادغام سنسور بیزی** — وقتی چندین سنسور یک شیء را تشخیص می‌دهند، نمرات اطمینان با استفاده از log-odds بیزی ترکیب می‌شوند — نه فقط "بالاترین را انتخاب کن". توافق دو سیگنال ضعیف می‌تواند بر یک سیگنال قوی غلبه کند.
- **سایه‌نماها و عکس‌های هواپیما** — بیش از ۱۲۰ کد نوع ICAO با ۱۰ دسته سایه‌نمای برداری (بدنه باریک، بدنه عریض، منطقه‌ای، توربوپراپ، جت تجاری، هلیکوپتر، جنگنده، باری، هواپیمای سبک، پهپاد) مطابقت داده شده و ۱۳۴ عکس هواپیمای بسته‌بندی شده برای تشخیص بصری فوری — بدون نیاز به شبکه.
- **نمای نقشه دوبعدی** — OpenStreetMap با شکل‌های نشانگر متمایز برای هر دسته، حلقه‌های فاصله، حالت دنبال کردن قطب‌نما و پوشش مخروط FOV.
- **کارت‌های جزئیات** — روی هر شیء ضربه بزنید برای جزئیات کامل: ثبت‌نام، اپراتور، مسیر (مبدأ/مقصد)، ارتفاع، سرعت، جهت، کد squawk، منبع تشخیص و سطح اطمینان.
- **گزارش تاریخچه** — پایگاه داده دائمی همه چیزهایی که شناسایی کرده‌اید، قابل جستجو و مرتب‌سازی.
- **راهنمای مرجع پهپاد** — پایگاه داده داخلی بیش از ۳۰ نوع پهپاد با عکس، مشخصات و توضیحات — از پهپادهای مصرف‌کننده DJI تا پهپادهای نظامی مانند Bayraktar TB2، MQ-9 Reaper و Shahed-136. بر اساس دسته مرور کنید، با نام جستجو کنید، یا مستقیماً از تشخیص پهپاد ناشناخته به شناسایی آنچه می‌بینید بروید.
- **غنی‌سازی بک‌اند (اختیاری)** — یک سرور Python API اختیاری می‌تواند نام شرکت‌های هواپیمایی، شماره‌های ثبت و اطلاعات مسیر فراتر از آنچه بسته‌بندی شده اضافه کند. اپلیکیشن بدون آن به طور کامل کار می‌کند — عکس‌های هواپیما و مراجع پهپاد در APK گنجانده شده‌اند.

## چگونه کار می‌کند

### منابع تشخیص

| منبع | چه چیزی تشخیص می‌دهد | برد | اطمینان |
|------|----------------------|-----|---------|
| **ADS-B** | هواپیماهای تجاری و GA با ترانسپوندر | ~۲۵۰ NM | بسیار بالا (۰.۹۵) |
| **Remote ID (BLE)** | پهپادهای سازگار با FAA (۲۵۰ گرم+) | ~۳۰۰ متر | بالا (۰.۸۵) |
| **WiFi SSID** | DJI، Skydio، Parrot، پهپادهای ارزان | ~۱۰۰ متر | متوسط (۰.۳-۰.۸۵) |
| **بصری (ML Kit)** | هر چیز قابل مشاهده در دوربین | خط دید | متغیر |

### ادغام سنسور و AR

اپلیکیشن داده‌های شتاب‌سنج، مغناطیس‌سنج و ژیروسکوپ را ادغام می‌کند تا دقیقاً تعیین کند گوشی به کجا اشاره می‌کند. موقعیت هواپیماها (عرض/طول/ارتفاع) با استفاده از محاسبات فاصله haversine و هندسه FOV دوربین روی نمای دوربین تصویر می‌شوند.

**ترکیبی ARCore + قطب‌نما**: ARCore ردیابی عالی ارائه می‌دهد وقتی دوربین ویژگی‌های زمینی را می‌بیند، اما وقتی به سمت آسمان بدون ویژگی نشانه‌گیری می‌شود مشکل دارد — دقیقاً زمانی که بیشترین نیاز به آن دارید. اپلیکیشن وقتی ARCore ردیابی را از دست می‌دهد به طور خودکار به جهت‌یابی ریاضی قطب‌نما برمی‌گردد و برچسب‌های یکپارچه در همه شرایط ارائه می‌دهد.

## شروع کار

### پیش‌نیازها

- **Android Studio** Hedgehog (2023.1.1) یا جدیدتر
- **JDK 17**
- **Android SDK 35** (compileSdk) با minSdk 26 (Android 8.0+)
- یک دستگاه اندروید با GPS، قطب‌نما و دوربین (شبیه‌ساز برای نماهای لیست/نقشه کار می‌کند اما نه AR)

> **نیازی به حساب یا ثبت‌نام نیست.** اپلیکیشن مستقیماً به API های عمومی رایگان ADS-B متصل می‌شود — بدون راه‌اندازی بک‌اند، بدون کلید API، بدون حساب.

### شروع سریع — فقط اندروید

اپلیکیشن مستقیماً به API های عمومی رایگان ADS-B متصل می‌شود — بدون راه‌اندازی بک‌اند یا حساب. عکس‌های هواپیما و راهنمای مرجع پهپاد در APK بسته‌بندی شده‌اند. برای ساخت و اجرا:

```bash
cd android
./gradlew assembleDebug
# نصب روی دستگاه متصل:
adb install app/build/outputs/apk/debug/app-debug.apk
```

یا آخرین APK پیش‌ساخته را از [**GitHub Releases**](https://github.com/lnxgod/friendorfoe/releases) دانلود کنید.

### راه‌اندازی بک‌اند (اختیاری — غنی‌سازی را فعال می‌کند)

بک‌اند عکس‌های هواپیما، نام شرکت‌های هواپیمایی، جستجوی ثبت‌نام و اطلاعات مسیر اضافه می‌کند. **به Python 3.11+ و اختیاراً Redis برای کش نیاز دارد.**

```bash
cd backend

# ایجاد محیط مجازی
python -m venv .venv
source .venv/bin/activate  # یا در ویندوز .venv\Scripts\activate

# نصب وابستگی‌ها
pip install -r requirements.txt

# کپی پیکربندی محیط
cp .env.example .env
# اگر می‌خواهید اعتبارنامه OpenSky اضافه کنید .env را ویرایش کنید (اختیاری)

# اجرای سرور
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**با Docker** (شامل Redis + PostgreSQL):

```bash
cd backend
docker compose up
```

API در `http://localhost:8000` در دسترس خواهد بود. بررسی سلامت: `http://localhost:8000/health`

### اتصال اپلیکیشن به بک‌اند

URL بک‌اند را در پیکربندی شبکه اپلیکیشن اندروید به‌روزرسانی کنید تا به آدرس IP سرور شما اشاره کند.

**Vibe coding چیست؟** این یک رویکرد مشارکتی و مکالمه‌ای به توسعه نرم‌افزار است که در آن انسان و هوش مصنوعی در زمان واقعی با هم می‌سازند. به جای نوشتن هر خط با دست، شما توصیف می‌کنید که چه می‌خواهید، روی ایده‌ها تکرار می‌کنید، با هم دیباگ می‌کنید و اجازه می‌دهید AI کد الگویی را مدیریت کند در حالی که شما روی چشم‌انداز و معماری تمرکز می‌کنید. این برنامه‌نویسی با حس است — و کار می‌کند.

Friend or Foe با چندین AI ساخته شد، هر کدام جایی که بهتر بود مشارکت کرد. Claude به عنوان شریک اصلی کدنویسی خدمت کرد — از معماری اولیه تا الگوریتم‌های ادغام سنسور، ریاضیات بیزی و هنر drawable برداری. Grok به شکل‌دهی جهت طراحی کمک کرد. Codex ممیزی امنیتی را قبل از انتشار متن‌باز انجام داد. Gemini تحقیق کرد که کدام فناوری‌ها و فریم‌ورک‌ها استفاده شوند. و ML Kit روی دستگاه اجرا می‌شود و بدون هیچ وابستگی ابری تشخیص بصری را تأمین می‌کند. هر فایل در این مخزن با همکاری انسان-AI شکل گرفت.

**ما باور داریم که آینده توسعه نرم‌افزار استفاده از AI مناسب برای کار مناسب است** و ما این پروژه را به صورت متن‌باز منتشر می‌کنیم تا شما ببینید این در عمل چگونه به نظر می‌رسد.

## درباره GAMECHANGERSai

**[GAMECHANGERSai](https://gamechangersai.org)** یک سازمان غیرانتفاعی 501(c)(3) است که به ایجاد تجربیات یادگیری عملی مبتنی بر AI برای کودکان و خانواده‌ها اختصاص دارد.

> *"بازی‌های AI سرگرم‌کننده که به هر یادگیرنده ساختن و بازسازی را آموزش می‌دهند."*

از طریق پلتفرم ما، یادگیرندگان مهارت‌های واقعی و قابل انتقال توسعه می‌دهند:

- **کدنویسی خلاقانه** — ساخت برنامه‌های کاربردی با AI به عنوان کمک‌خلبان
- **تفکر سیستمی** — درک نحوه تعامل بخش‌های پیچیده و نحوه رفع اشکال آن‌ها
- **مدیریت منابع** — برنامه‌ریزی استراتژیک و بهینه‌سازی
- **استدلال اخلاقی** — یادگیری پرسیدن "آیا باید؟" در کنار "آیا می‌توانیم؟" هنگام استفاده از AI

**Friend or Foe اثبات می‌کند که این مهارت‌ها واقعی هستند.** همان کدنویسی خلاقانه، تفکر سیستمی و همکاری با AI که بچه‌ها در پلتفرم ما یاد می‌گیرند برای ساخت این سیستم شناسایی هواپیمای کاملاً کاربردی استفاده شد. آنچه به عنوان بازی شروع می‌شود ابزار می‌شود. آنچه به عنوان سرگرمی شروع می‌شود تخصص می‌شود.

ما متعهد به بازگرداندن به جامعه متن‌باز هستیم. با انتشار عمومی Friend or Foe، امیدواریم:

- توسعه‌دهندگان و یادگیرندگان را **الهام** دهیم تا کشف کنند چه چیزی با توسعه به کمک AI ممکن است
- **نشان** دهیم که AI یک شریک خلاق است، نه جایگزین — هر تصمیم در این اپلیکیشن با قضاوت انسانی هدایت شد
- **نشان** دهیم که مهارت‌هایی که بچه‌های ما یاد می‌گیرند فقط برای بازی نیست — آن‌ها پایه ساخت فناوری واقعی و مفید هستند

برای آشنایی بیشتر با مأموریت ما به **[gamechangersai.org](https://gamechangersai.org)** مراجعه کنید.

## منابع داده و ارجاع

استفاده از Friend or Foe ۱۰۰٪ رایگان است — هر منبع داده زیر باز است و نیاز به **کلید API یا حساب پولی** ندارد.

### داده‌های موقعیت هواپیما ADS-B

اپلیکیشن از زنجیره پشتیبان سه لایه‌ای استفاده می‌کند تا همیشه داده هواپیما دریافت کنید حتی اگر یک منبع از کار بیفتد:

| اولویت | منبع | نقطه پایانی API | چه چیزی ارائه می‌دهد | احراز هویت |
|--------|------|----------------|---------------------|-----------|
| اول | **[adsb.fi](https://adsb.fi)** | `opendata.adsb.fi/api` | موقعیت هواپیما در زمان واقعی، علامت‌های تماس، ارتفاع، سرعت، جهت، کدهای squawk | رایگان، بدون کلید |
| دوم | **[airplanes.live](https://airplanes.live)** | `api.airplanes.live` | همان داده‌های adsb.fi (فرمت سازگار ADSBx v2) | رایگان، بدون کلید |
| سوم | **[OpenSky Network](https://opensky-network.org)** | `opensky-network.org/api` | بردارهای وضعیت ICAO — موقعیت، سرعت، جهت، نرخ عمودی | رایگان، بدون کلید (حساب اختیاری برای محدودیت‌های نرخ بالاتر) |

هر سه منبع داده‌های زنده ترانسپوندر ADS-B از شبکه‌های گیرنده در سراسر جهان ارائه می‌دهند. اپلیکیشن بر اساس مختصات GPS و شعاع پرس‌وجو می‌کند و در صورت شکست یا اتمام وقت به طور خودکار به منبع بعدی می‌رود.

### غنی‌سازی و داده‌های پشتیبان

| منبع | چه چیزی ارائه می‌دهد | احراز هویت |
|------|---------------------|-----------|
| **[Planespotters.net](https://planespotters.net)** | عکس‌های هواپیما با کد ICAO hex — در کارت‌های جزئیات نمایش داده می‌شوند | رایگان، بدون کلید |
| **[Open-Meteo](https://open-meteo.com)** | آب‌و‌هوای فعلی (پوشش ابر، سرعت باد، شرایط) — برای تنظیم پارامترهای تشخیص استفاده می‌شود | رایگان، بدون کلید |
| **[OpenStreetMap](https://openstreetmap.org)** | کاشی‌های نقشه از طریق OSMDroid برای نمای نقشه دوبعدی | رایگان، بدون کلید |

### تشخیص روی دستگاه (بدون API خارجی)

این روش‌های تشخیص کاملاً روی گوشی بدون تماس شبکه‌ای اجرا می‌شوند:

- **FAA Remote ID (Bluetooth LE)** — پخش‌های پهپاد سازگار را در محدوده ~۳۰۰ متر اسکن می‌کند
- **تطابق WiFi SSID** — DJI، Skydio، Parrot و بیش از ۱۰۰ سازنده پهپاد دیگر را با الگوهای WiFi شناسایی می‌کند
- **تشخیص بصری (ML Kit)** — تشخیص اشیا مبتنی بر دوربین روی دستگاه
- **طبقه‌بندی نظامی** — الگوهای علامت تماس، کدهای squawk و پایگاه داده اپراتور (همه به صورت محلی بسته‌بندی شده)
